# Decisions Log (ADR-lite)

Sem zapisovat rozhodnutí, která mění implementaci oproti draftům nebo doplňují chybějící detaily.

## Kdy zapisovat

- změna UX patternu
- změna motion chování
- technický kompromis (např. vypnutí Lenis na mobile)
- odchylka od styleguide
- doplněný předpoklad kvůli chybějícím podkladům

## Naming convention

- `YYYY-MM-DD-short-title.md`

## Template

```md
# Decision: <název>

## Date
YYYY-MM-DD

## Context
Jaký problém nebo nejasnost se řeší.

## Decision
Co bylo rozhodnuto.

## Impact
Jaký to má dopad na UX, frontend, výkon, a11y nebo další tasky.

## Alternatives considered
Krátce: co se zvažovalo a proč ne.
```
